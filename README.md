## Описание

В ходе выполнения лабораторной работы были изучены базовые конструкции языка JavaScript. Основное внимание уделено следующим темам:

- **Типы данных:** Работа с примитивами (`number`, `string`, `boolean`, `bigint`, `symbol`) и специальными значениями (`null`, `undefined`).
- **Динамическая типизация:** Изучена возможность смены типа переменной в процессе выполнения кода.
- **Преобразование типов:** Рассмотрены явные (функции `Number`, `String`, `Boolean`) и неявные преобразования при математических операциях.
- **Операторы сравнения:** Разница между нестрогим (`==`) и строгим (`===`) равенством.
- **Управляющие конструкции:** Использование условий `if/else`, тернарного оператора и многовариантного выбора `switch`.
- **Сложные типы:** Работа с объектами (литеральная нотация) и массивами.

## Итоговая таблица: JavaScript vs C#

| Аспект              | C#                           | JavaScript                    |
| :------------------ | :--------------------------- | :---------------------------- |
| **Типизация**       | Статическая                  | Динамическая                  |
| **Компиляция**      | Да (в IL-код)                | Нет (интерпретация/JIT)       |
| **Точка входа**     | `Main()`                     | Отсутствует                   |
| **Переменные**      | `int x = 5;`                 | `let x = 5;`                  |
| **Константы**       | `const int X = 5;`           | `const X = 5;`                |
| **Вывод в консоль** | `Console.WriteLine()`        | `console.log()`               |
| **Проверка типа**   | Во время компиляции          | `typeof`                      |
| **Сравнение**       | `==` (по значению)           | `===` (строгое)               |
| **Условия**         | `if/else/switch`             | `if/else/switch` + тернарный  |
| **Циклы**           | `for/while/do-while/foreach` | `for/while/do-while/for...of` |
| **Функции**         | Методы с типами              | Функции без типов             |
| **Массивы**         | `int[] arr = new int[5];`    | `let arr = [];`               |
| **Объекты**         | Требуют класс                | Можно создать литералом       |
| **ООП**             | Классическое                 | Прототипное                   |
| **Null**            | `null`                       | `null` и `undefined`          |
| **Строгость**       | Высокая (компилятор)         | Низкая (нужна дисциплина)     |

## Главные выводы

1.  **JavaScript гибче C#** — можно свободно менять типы переменных в процессе работы.
2.  **JavaScript проще синтаксически** — отсутствует необходимость в строгом описании классов и типов для простых задач.
3.  **JavaScript опаснее** — ошибки часто обнаруживаются только во время выполнения (runtime), а не на этапе написания.
4.  **C# безопаснее** — строгий компилятор предотвращает множество ошибок еще до запуска программы.
